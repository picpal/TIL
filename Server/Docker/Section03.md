> 데이터 카테고리, 다양한 종류의 데이터 이해하기


크게 3가지의 영역으로 구분된다

# (이미지 레이어) Application 
e.g) Code + Environment
- 이미지로 변환되면 고정된다. 
- 이미지는 읽기 전용이고 변경을 위해서는 다시 빌드해야한다.
- 기존 이미지를 변경 시작 할 수 없다.
- 읽기 전용 환경
- 실행중인 어플리케이션을 변경하는것 안된다

# (컨테이너 레이어) 임시 App Data
e.g) entered user input
- 사용자가 입력한 값을 서버에서 처리하는데 이과정에서 서버 메모리나 임시적인 파일에 저장한다
- 이런경우 서버를 멈추게 되면 잃어버리게 되는 값이다
- 읽기, 쓰기 

# (볼륨 레이어) 영구적인 App Data
e.g) user accounts
- 컨테이너가 중지되고 재시작 되더라도 남아 있어야
